# Суть API пакета `db`
Пакет `db` предоставляет функционал для работы с базой данных PostgreSQL, включая сохранение версионированных JSON-данных, создание аналитических представлений (Views) и экспорт отчетов в формате Excel (XLSX).

# Установка API пакета `db`
```bash
go get github.com/tfffr/db
```

# Документация API пакета `db`

## `NewConnection`

Создаёт новое подключение к PostgreSQL и возвращает инициализированный экземпляр `DBConnection`.

### Сигнатура

```go
func NewConnection(dsn, table, dateTable string) (*DBConnection, error)
```

### Параметры

* `dsn` — строка подключения к PostgreSQL.
* `table` — имя основной таблицы, в которой хранятся JSON-данные.
* `dateTable` — имя таблицы, в которой хранятся даты фиксации записей.

### Возвращает

* `*DBConnection` — объект подключения с сохранёнными именами таблиц.
* `error` — ошибка открытия соединения или недоступности БД.

### Поведение

* Открывает соединение через драйвер `postgres`.
* Выполняет `Ping`, чтобы убедиться, что БД доступна.
* Не создаёт таблицы автоматически.

### Возможные ошибки

* ошибка драйвера БД;
* БД недоступна.

### Пример

```go
dsn := "host=localhost port=5432 user=user password=pswd dbname=db sslmode=disable"

conn, err := NewConnection(dsn, "records", "records_dt")
if err != nil {
	panic(err)
}
defer conn.Close()
```

## `(*DBConnection).Close`

Закрывает активное соединение с базой данных.

### Сигнатура

```go
func (conn *DBConnection) Close() error
```

### Возвращает

* `error` — ошибка закрытия соединения, если возникла.

### Пример

```go
if err := conn.Close(); err != nil {
	panic(err)
}
```

## `(*DBConnection).InitTables`

Создаёт основную таблицу и таблицу дат, если они ещё не существуют.

### Сигнатура

```go
func (conn *DBConnection) InitTables() error
```

### Поведение

Метод создаёт:

1. основную таблицу со столбцами:

   * `id`
   * `json_data`
   * `hash`
   * `page_number`

2. таблицу дат со столбцами:

   * `id`
   * `main_id`
   * `dt`

### Особенности

* используется SQL-шаблон `SQLCreateTable`;
* таблица дат связывается с основной таблицей через `main_id`;
* при удалении записи из основной таблицы связанные даты удаляются каскадно.

### Возвращает

* `error` — ошибка выполнения SQL-запроса.

### Пример

```go
if err := conn.InitTables(); err != nil {
	panic(err)
}
```

## `(*DBConnection).SaveBatch`

Сохраняет пакет JSON-записей в основную таблицу и фиксирует дату их появления в таблице дат.

### Сигнатура

```go
func (conn *DBConnection) SaveBatch(items []json.RawMessage, pageNumber int) error
```

### Параметры

* `items` — массив JSON-объектов для сохранения.
* `pageNumber` — номер страницы/пакета, связанный с сохраняемыми данными.

### Поведение

* Если `items` пустой, метод завершится без ошибки.
* Для каждой записи:

  * вычисляется SHA-256 хэш;
  * выполняется вставка в основную таблицу;
  * при конфликте по `hash` запись не дублируется;
  * возвращается `id` записи;
  * в таблицу дат добавляется новая запись с этим `id`.
* Все операции выполняются в одной транзакции.

### Возвращает

* `error` — ошибка начала транзакции, вставки данных, вставки даты или коммита.

### Особенности

* дедупликация выполняется по SHA-256 хэшу JSON-данных;
* даже если запись уже существует, дата фиксации добавляется отдельно;
* при любой ошибке транзакция откатывается.

### Пример

```go
items := []json.RawMessage{
	json.RawMessage(`{"id":1,"name":"Alice"}`),
	json.RawMessage(`{"id":2,"name":"Bob"}`),
}

if err := conn.SaveBatch(items, 1); err != nil {
	panic(err)
}
```

## `(*DBConnection).CreateOutputView`

Создаёт или обновляет SQL `VIEW`, содержащий итоговое представление по включению и исключению записей.

### Сигнатура

```go
func (conn *DBConnection) CreateOutputView(viewName string) error
```

### Параметры

* `viewName` — имя создаваемого представления.

### Поведение

Во view рассчитываются:

* `json_data` — исходные данные записи;
* `dt_inclusion` — дата первого появления записи в последнем непрерывном интервале присутствия;
* `dt_exclusion` — дата следующего дампа после последнего появления записи в этом интервале.

### Логика

Метод использует цепочку CTE:

* собирает все уникальные даты дампов;
* упорядочивает их;
* определяет присутствие записей в конкретные даты;
* находит непрерывные интервалы присутствия;
* выбирает последний актуальный интервал для каждой записи.

### Возвращает

* `error` — ошибка создания view.

### Пример

```go
if err := conn.CreateOutputView("v_output"); err != nil {
	panic(err)
}
```

## `(*DBConnection).ExportViewToExcel`

Экспортирует данные из указанного view в XLSX-файл.

### Сигнатура

```go
func (conn *DBConnection) ExportViewToExcel(viewName string, fileName string) error
```

### Параметры

* `viewName` — имя view, из которого читаются данные.
* `fileName` — имя выходного XLSX-файла.

### Поведение

Метод:

1. выполняет запрос к view;
2. создаёт новый Excel-файл;
3. создаёт лист с именем, заданным в `SheetName`;
4. записывает заголовки:

   * `JSON`
   * `Дата включения записи`
   * `Дата исключения записи`
5. построчно заполняет файл данными;
6. сохраняет файл на диск.

### Формат выходного файла

| Колонка | Содержимое      |
| ------- | --------------- |
| A       | JSON-данные     |
| B       | дата включения  |
| C       | дата исключения |

### Возвращает

* `error` — ошибка запроса к view, создания листа, чтения строк или сохранения файла.

### Пример

```go
if err := conn.ExportViewToExcel("v_output", "report.xlsx"); err != nil {
	panic(err)
}
```

## `SetupTestDB`

Подготавливает тестовую базу данных для интеграционных тестов.

> Функция предназначена для тестового окружения.

### Сигнатура

```go
func SetupTestDB(t *testing.T) *DBConnection
```

### Параметры

* `t` — объект теста.

### Поведение

* создаёт подключение к тестовой БД;
* удаляет тестовые таблицы, если они существуют;
* заново инициализирует структуру таблиц;
* возвращает готовое подключение.

### Особенности

* при ошибке завершает тест через `t.Fatalf`;
* использует константы подключения:

  * `DBHost`
  * `DBPort`
  * `DBUser`
  * `DBPswd`
  * `DBName`

### Пример

```go
conn := SetupTestDB(t)
defer conn.Close()
```

## `FillTestData`

Заполняет тестовую БД предопределённым набором записей и дат.

> Функция предназначена для тестового окружения.

### Сигнатура

```go
func FillTestData(t *testing.T, conn *DBConnection)
```

### Параметры

* `t` — объект теста.
* `conn` — тестовое подключение к БД.

### Поведение

Функция вставляет три тестовые записи и моделирует изменения между несколькими дампами:

* часть записей появляется в первом дампе;
* часть исчезает во втором;
* одна из записей появляется повторно в следующем дампе.

Это используется для проверки корректности вычисления:

* даты включения;
* даты исключения;
* повторного появления записи.

### Пример

```go
conn := SetupTestDB(t)
defer conn.Close()

FillTestData(t, conn)
```
